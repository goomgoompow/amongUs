# 브라우저 기반 소셜 디덕션 게임

개인 PC에서 서버를 실행하고, 플레이어가 웹 브라우저로 접속하는 실시간 멀티플레이 게임 프로젝트입니다.

현재 단계에서는 구현 전에 합의할 기준 구조를 정리했습니다.

- [전체 시스템 설계](docs/architecture.md)
- [MVP 개발 순서](docs/architecture.md#mvp-개발-순서)
- [실시간 메시지 초안](docs/architecture.md#실시간-메시지-초안)

이 프로젝트는 특정 게임의 이름·맵·캐릭터·그래픽을 복제하지 않고, 소셜 디덕션 장르의 핵심 재미를 바탕으로 독자적인 테마와 규칙을 만드는 것을 전제로 합니다.

## 현재 구현

- 방 생성과 6자리 초대 코드
- 최대 10명 참가 및 실시간 참가자 목록
- 방장 표시와 방장 퇴장 시 자동 이전
- 준비/준비 취소 및 게임 시작 가능 조건
- 데스크톱·모바일 대응 대기실 화면
- 4~10명 목표 인원과 1~2명 임포스터 설정
- 부족한 자리를 컴퓨터 대원으로 자동 충원
- 비공개 역할 배정과 테스트 맵 입장
- WASD/방향키 이동과 실시간 위치 동기화
- 컴퓨터 대원의 자동 이동과 임무 수행
- 공동 임무 진행도 및 시민 승리 판정
- 30,000 x 20,000 대형 월드와 15개 주요 구역
- 플레이어 중심 추적 카메라와 주변 객체만 표시하는 범위 렌더링
- 현재 좌표와 전체 월드 미니맵 위치 표시
- 방과 복도 외부 이동을 막는 서버 충돌 판정
- CPU 방 연결 그래프 경로 탐색
- 가까운 문 열기와 임포스터 문 잠금
- 주변 문 5개를 폐쇄하는 임포스터 방해 공작
- 문 자동 해제와 방해 공작 쿨다운
- 역할별 시야 거리와 벽·닫힌 문 기반 시야 차단
- 시야 밖 플레이어 위치를 클라이언트에 보내지 않는 서버 필터링
- 게임 시작 시 전체 화면 역할 공개와 지속 역할 HUD
- 임포스터 전용 문 잠금 권한
- 거리·시야·생존 상태를 검증하는 크루 제거 공격
- 공격 쿨다운, 시체 표시 및 임포스터 인원 승리 판정
- 이동 방향을 바라보는 다리·산소통 포함 우주복 캐릭터
- 사람과 CPU의 완화된 이동속도
- 승리 화면에서 같은 방과 설정으로 다시 시작하는 방장 재경기 기능

## 공동 작업 환경

1. Node.js LTS를 설치합니다.
2. 저장소를 복제한 폴더에서 `npm start`를 실행합니다.
3. 브라우저에서 `http://localhost:3000`을 엽니다.

Windows에서는 `start-game.cmd`를 더블클릭해도 됩니다. 같은 Wi-Fi에 연결된 다른 기기는 서버 PC의 내부 IP 뒤에 `:3000`을 붙여 접속할 수 있습니다.
